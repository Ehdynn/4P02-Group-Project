  An open source alternative to TurnitIn that is speciallized for detecting plagiarism in code. Supports Java, C, C++, and Python. 
 

  Professors may run multiple classes each with multiple assignments for ease of use. 

  Boilerplate Code may be included by the professor when creating an assignment, which will prevent false positive plagiarism detection on professor provided code (this may be an interface or abstract class students are expected to create an implementation for, a given function, etc.).

  Student information is securely handled to respect privacy. 

  Plagiarism reports will highlight all plagiarized segments of code, and will colorcode the segments based on the source of the segment, and the overall score of the plagiarism. If the proffessor clicks a highlighted segment, it will pull up the other source code side by side with the current one to make comparison easier. 

  Professors will be able to compare the students' submissions to all other submissions in their classes, alongside any given database of code the professor uploads. This will enable professors to keep their previous history of assignments from other programs and use it here. 

  Professors will be able to create an account with no hassle. It is as easy as setting up an email account, and picking a password. Professors will also be able to host the website themselves to be in compliance with their schools data privacy policies. 
  
  Students will be able to easily upload their code for plagiarism detection through an easy to use web portal. All the student will need is the course code, and their zipped submission. No account needed. 

  To find more information on how to use and operate this project, head over to the documentation directory.
